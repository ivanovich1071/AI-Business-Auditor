## 1. Архитектура приложения

### Общая схема работы
1. Пользователь вводит URL сайта → Frontend отправляет запрос на бэкенд.
2. Бэкенд:
   - Парсит максимальное количество страниц сайта (игнорируя страницы с ошибками, например 404).
   - Подключается к бесплатным MCP-сервисам для сбора дополнительной информации о компании из открытых источников.
   - Анализирует текст с помощью Qwen 3.7 для определения отрасли, бизнес-процессов и болей.
   - Генерирует список из 5–10 AI-агентов, ранжированных по степени полезности, с объяснением, почему они подходят.
3. Результаты отображаются в области анализа с анимацией загрузки (прогресс-бар).
4. Пользователь может обсудить результаты в чате (тематика строго ограничена вопросами по анализам).
5. Результаты сохраняются в дашборде (с возможностью удаления записей).

### Компоненты архитектуры

| Компонент | Описание | Технологии |
|---|---|---|
| Frontend | Одноколоночный интерфейс с полем для URL, кнопкой анализа, областью результатов и чатом | Next.js 14+, TypeScript, Tailwind CSS |
| Backend | Обрабатывает запросы, парсит сайты, взаимодействует с MCP и AI-моделями | Next.js (API Routes), Prisma, SQLite |
| MCP-сервисы | Бесплатные сервисы для сбора данных о компании из открытых источников | API MCP (SerpAPI, DuckDuckGo, OpenCorporates) |
| AI-модели | Анализ текста сайта и генерация предложений по AI-агентам | Qwen 3.7 (OpenRouter) |
| База данных | Хранение результатов анализов (без истории чатов) | SQLite + Prisma ORM |
| Парсинг сайтов | Скачивание и анализ HTML-страниц (максимальное количество страниц, игнорируя ошибки) | Cheerio, Puppeteer (для SPA) |

## 2. Интерфейс приложения (UI/UX)

### Макет и дизайн
- **Цветовая палитра:** фон `#FAF9F6`, текст `#2C1B18`, акценты `#3E2723`, `#D4AF37`/`#FFC107`, тени `shadow-lg` тёплого оттенка.
- **Типографика:** заголовки `text-2xl`/`text-3xl`, основной текст `text-lg`, шрифты Inter/Manrope.
- **Скругления:** `rounded-2xl`/`rounded-3xl`.
- **Анимации:** плавные переходы `transition-all duration-300`, прогресс-бар при загрузке, спиннеры в фирменных цветах.

### Структура интерфейса (одноколоночный макет)
```
┌───────────────────────────────────────────────────────┐
│  Header: Лого + Название "AI Business Auditor"         │
├───────────────────────────────────────────────────────┤
│  Поле для URL (иконка поиска)                           │
│  Кнопка «Проанализировать»                              │
│  Прогресс-бар при загрузке                              │
├───────────────────────────────────────────────────────┤
│  Область результатов (скроллится независимо)           │
│  - Карточки AI-агентов (ранжированные)                  │
│  - Описание компании и отрасли                           │
│  - Объяснение, почему предложены именно эти агенты      │
│  - Кнопка «Сохранить в дашборд»                          │
├───────────────────────────────────────────────────────┤
│  Чат с AI (тематика строго ограничена анализами)        │
│  - История диалога (скроллится, только для сессии)      │
│  - Поле ввода сообщения + кнопка отправки                │
├───────────────────────────────────────────────────────┤
│  Footer: ссылка на дашборд, копирайт © 2026              │
└───────────────────────────────────────────────────────┘
```

### Вкладка "Дашборд"
- Список анализов (карточки): название компании, URL, дата, отрасль, количество агентов.
- Кнопки «Открыть», «Удалить».
- Фильтры: поиск по названию компании.
- Статистика: частота агентов, распределение по отраслям.

## 6. Обработка ошибок

| Тип ошибки | Действие |
|---|---|
| Сайт недоступен | «Сайт недоступен. Проверьте URL или попробуйте позже.» |
| Ошибка парсинга (404) | Игнорируется, анализ продолжается без этой страницы |
| Ошибка MCP-сервиса | «Не удалось получить дополнительные данные. Используем только информацию с сайта.» |
| Ошибка AI-модели | «Извините, произошла ошибка. Попробуйте ещё раз.» |

---

## 7. Безопасность и конфиденциальность

- Проверка вводимых URL (список блокировки вредоносных сайтов).
- Rate limiting: 10 запросов в минуту с одного IP.
- Не сохранять личные данные пользователей и историю чатов после сессии.
- API-ключи (OpenRouter, MCP) — только в переменных окружения (`.env`).

## 8. Сценарии работы (примеры)

**Сценарий A — успешный анализ:** URL → парсинг главной/«О нас»/«Продукты»/«Команда» →
SerpAPI + OpenCorporates → AI определяет отрасль и бизнес-процессы → карточки агентов
с рангом и объяснением → чат.

**Сценарий B — сайт недоступен:** 404 → уведомление → повторный ввод URL.

**Сценарий C — MCP недоступен:** сайт парсится успешно, MCP падает → уведомление,
анализ продолжается на основе только текста сайта.
